Generative tables transform structured data from backend APIs into rich, interactive table components that adapt to the tool being executed and the data being displayed.

## Overview

The `DataResultCard` component is the primary generative table implementation in Gima AI. It receives paginated data from backend tools and automatically:

- Selects optimal columns based on the tool type
- Applies context-aware formatting and badges
- Provides collapsible UI with pagination info
- Highlights critical data (low stock, priority levels, status)

## Architecture

### Backend Integration

Generative tables consume data from chat tools defined in `/app/lib/ai/tools/chat-tools.ts`. Each tool returns:

```typescript
interface ToolResponse {
  success: boolean;
  data: {
    items: Record<string, unknown>[];
    pagination: {
      page: number;
      lastPage: number;
      total: number;
      hasMore: boolean;
    };
  };
  summary?: string;
}
```

### Stock Indicators

Inventory tables display color-coded stock levels:

- **Critical**: Red (stock ≤ minimum)
- **Warning**: Yellow (stock ≤ minimum × 1.5)
- **Normal**: Default color

```typescript
function StockIndicator({ stock, stockMin }) {
  const isCritical = stockMin && stock <= stockMin;
  const isWarning = stockMin && stock <= stockMin * 1.5;
  // Returns formatted value with color class
}
```

## Component Usage

### Basic Table

```tsx
import { DataResultCard } from '@/app/components/features/chat/tool-result-cards';

function ChatMessage({ toolResult }) {
  return (
    <DataResultCard
      data={{
        items: toolResult.items,
        pagination: toolResult.pagination,
      }}
      toolName="tool-consultar_activos"
      summary="Found 25 assets in the system"
    />
  );
}
```

## Custom Rendering

Column definitions support custom render functions:

```typescript
interface ColumnDef {
  key: string;
  label: string;
  render?: (value: unknown, item: Record<string, unknown>) => ReactNode;
}

// Example: Custom badge rendering
{
  key: 'estado',
  label: 'Estado',
  render: (v) => v ? <StatusBadge value={String(v)} /> : '—',
}

// Example: Nested data access
{
  key: 'sede',
  label: 'Ubicación',
  render: (_, item) => {
    const ubic = item.ubicacion as Record<string, unknown>;
    return [ubic?.edificio, ubic?.salon].filter(Boolean).join(' - ') || '—';
  },
}
```

## Data Formatting

### Cell Value Formatting

```typescript
function formatCellValue(value: unknown): string {
  if (value === null || value === undefined) return '—';
  if (typeof value === 'boolean') return value ? 'Sí' : 'No';
  if (typeof value === 'number') return value.toLocaleString('es-VE');
  return String(value);
}
```

### Column Header Formatting

```typescript
function formatColumnHeader(key: string): string {
  return key
    .replace(/_/g, ' ')
    .replace(/\b\w/g, c => c.toUpperCase());
}
```

## Interaction Features

### Collapsible UI

Tables can be expanded or collapsed:

```tsx
const [expanded, setExpanded] = useState(true);

<button onClick={() => setExpanded(!expanded)}>
  {expanded ? <ChevronUp /> : <ChevronDown />}
</button>
```

### Pagination Display

```tsx
{data.pagination.lastPage > 1 && (
  <span className="text-xs text-muted-foreground">
    Pág. {data.pagination.page}/{data.pagination.lastPage}
  </span>
)}
```

## Performance

The backend limits results to prevent token overflow:

```typescript
// From chat-tools.ts
const MAX_ITEMS_PER_RESPONSE = 15;
const MAX_DESCRIPTION_LENGTH = 120;

const items = result.items.slice(0, MAX_ITEMS_PER_RESPONSE);
```

<Note>
  Large datasets are truncated server-side. Use pagination or filters to access more results.
</Note>
