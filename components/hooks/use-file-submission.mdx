## Overview

The `useFileSubmission` hook provides comprehensive file handling for chat messages, supporting automatic analysis of images and PDFs using AI.

### Features

- **Image Analysis** - Automatic visual analysis using Gemini Vision API
- **PDF Processing** - Extract and analyze PDF content
- **File Validation** - Size limits and format checking
- **Voice Integration** - Automatically stops voice recording when submitting
- **Progress Tracking** - Real-time analysis state indicators
- **Error Handling** - User-friendly error messages

## Import

```tsx
import { useFileSubmission } from '@/app/components/features/chat/hooks';
```

## How It Works

### File Detection

The hook automatically detects attached file types:

1. **Images** - Any file with `mediaType` starting with `image/`
2. **PDFs** - Files with `mediaType` of `application/pdf`
3. **Other** - Treated as normal text message

### Processing Flow

#### Image Files

1. Validates file size (max 5MB)
2. Converts blob URL to base64
3. Adds user message with image to chat
4. Calls `analyzePartImage` server action
5. Returns structured analysis:
   - Type of item detected
   - Physical condition
   - Confidence level
   - Brand/model (if applicable)
   - Quantity detected
   - Detailed description
   - Recommendations

#### PDF Files

1. Validates file size (max 10MB)
2. Converts blob URL to base64
3. Adds user message with PDF to chat
4. Calls `analyzePdf` server action
5. Returns text analysis of PDF content

#### Text Messages

1. Adds user message to chat
2. Calls `sendMessage` with configured model
3. Streams AI response

### File Size Limits

| File Type | Max Size |
|-----------|----------|
| Images | 5 MB |
| PDFs | 10 MB |

<Warning>
Files exceeding these limits will trigger an error and not be processed.
</Warning>

```tsx
function ChatWithFiles() {
```

### With File Upload

```tsx
import { useFileSubmission } from '@/app/components/features/chat/hooks';
import { useState } from 'react';

function FileUploadChat() {
  const [messages, setMessages] = useState<UIMessage[]>([]);
  const [isListening, setIsListening] = useState(false);
  const [selectedFile, setSelectedFile] = useState<File | null>(null);
  const [fileUrl, setFileUrl] = useState<string | null>(null);

  const { handleSubmit, isAnalyzing, analyzingFileType } = useFileSubmission({
    setMessages,
    sendMessage: async (msg, opts) => {
      // Send to your API
      return 'msg-id';
    },
    isListening,
    toggleListening: () => setIsListening(!isListening)
  });

  const handleFileSelect = (e: React.ChangeEvent<HTMLInputElement>) => {
    const file = e.target.files?.[0];
    if (file) {
      setSelectedFile(file);
      setFileUrl(URL.createObjectURL(file));
    }
  };

  const handleFormSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    
    if (!fileUrl || !selectedFile) return;

    await handleSubmit({
      text: 'Analyze this file',
      files: [{
        url: fileUrl,
        mediaType: selectedFile.type,
        name: selectedFile.name
      }]
    });

    // Cleanup
    setSelectedFile(null);
    setFileUrl(null);
  };

  return (
    <form onSubmit={handleFormSubmit}>
      <input
        type="file"
        accept="image/*,application/pdf"
        onChange={handleFileSelect}
        disabled={isAnalyzing}
      />

      {selectedFile && (
        <div className="preview">
          <p>Selected: {selectedFile.name}</p>
          {selectedFile.type.startsWith('image/') && (
            <img src={fileUrl!} alt="Preview" />
          )}
        </div>
      )}

      <button type="submit" disabled={!selectedFile || isAnalyzing}>
        {isAnalyzing ? `Analyzing ${analyzingFileType}...` : 'Submit'}
      </button>
    </form>
  );
}
```

## Error Handling

The hook provides comprehensive error handling:

```tsx
try {
  await handleSubmit({ text: 'Test', files });
} catch (error) {
  // Errors are automatically:
  // 1. Logged to console
  // 2. Added to chat as error message
  // 3. Shown via toast notification
  // 4. Reflected in isAnalyzing state
}
```

### Common Errors

- **File too large** - "El archivo excede el límite de XMB"
- **Invalid format** - "Formato de archivo no soportado"
- **Network error** - "Error de conexión"
- **API error** - "Error al analizar [tipo]: [detalle]"

## Performance Considerations

### Optimization Tips

1. **File Size** - Keep files under recommended limits
2. **Format** - Use JPEG for images (better compression than PNG)
3. **Concurrent Uploads** - Hook prevents concurrent analysis automatically
4. **Memory Management** - Blob URLs are created and managed efficiently
