## Overview

The `useVoiceInput` hook provides intelligent voice transcription with automatic mode switching between server-side (Gemini API) and client-side (Web Speech API) transcription.

### Features

- **Dual-mode transcription** - Primary: Gemini API for high accuracy, Fallback: Native Web Speech API
- **Automatic fallback** - Switches modes based on network status and API availability
- **User-friendly errors** - Simplifies technical errors into actionable messages
- **Browser compatibility** - Gracefully handles browser capabilities
- **Real-time feedback** - Provides state updates throughout the transcription process

## Import

```tsx
import { useVoiceInput } from '@/app/hooks/use-voice-input';
```

## How It Works

### Gemini Mode (Primary)

1. Requests microphone access
2. Records audio using MediaRecorder API
3. Detects supported audio format (webm/opus, webm/mp3, or audio/webm)
4. Converts audio to base64
5. Sends to Gemini API for transcription via server action
6. Returns high-accuracy transcription

### Native Mode (Fallback)

1. Uses browser's built-in Web Speech API
2. Real-time transcription as you speak
3. No server required
4. Lower accuracy than Gemini
5. Automatically activated when:
   - Gemini API is unavailable
   - Network is offline
   - Quota exceeded
   - MediaRecorder not supported

### Error Handling

The hook automatically simplifies technical errors:

| Error Type | User-Friendly Message |
|------------|----------------------|
| Quota exceeded | ⏸️ Límite de API alcanzado · Modo local activo |
| API key invalid | 🔑 API no configurada · Modo local activo |
| Network error | 📡 Sin conexión · Modo local activo |
| Timeout | ⏱️ Tiempo agotado · Modo local activo |
| Audio error | 🎤 Error de audio · Modo local activo |
| Permission denied | 🚫 Permiso de micrófono denegado |
| Not supported | ❌ Tu navegador no soporta reconocimiento de voz |

## Usage Examples

### Basic Usage

```tsx
import { useVoiceInput } from '@/app/hooks/use-voice-input';

function VoiceButton() {
  const {
    isListening,
    isProcessing,
    transcript,
    toggleListening,
    mode,
    error
  } = useVoiceInput({
    onTranscript: (text) => {
      console.log('Transcribed:', text);
    },
    onError: (error) => {
      console.error('Voice error:', error);
    },
    language: 'es-ES'
  });

  return (
    <div>
      <button 
        onClick={toggleListening} 
        disabled={isProcessing}
      >
        {isListening ? 'Stop' : 'Start'} ({mode})
      </button>
      
      {error && <p className="error">{error}</p>}
      {transcript && <p>{transcript}</p>}
      
      {isProcessing && <p>Processing...</p>}
    </div>
  );
}
```

### Chat Integration

```tsx
import { useVoiceInput } from '@/app/hooks/use-voice-input';
import { useState } from 'react';

function ChatWithVoice() {
  const [input, setInput] = useState('');
  const [status, setStatus] = useState<'listening' | 'processing' | 'idle'>('idle');

  const {
    isListening,
    isProcessing,
    toggleListening,
    mode,
    error
  } = useVoiceInput({
    onTranscript: (text) => {
      // Set transcribed text as input
      setInput(text);
    },
    onError: (error) => {
      console.error('Voice error:', error);
      // Still allow user to continue typing
    },
    onStateChange: (state) => {
      setStatus(state);
    },
    language: 'es-ES'
  });

  return (
    <div>
      <div className="status">
        {status === 'listening' && '🎤 Escuchando...'}
        {status === 'processing' && '⚙️ Procesando...'}
        {status === 'idle' && 'Listo'}
        {mode === 'native' && ' (Modo local)'}
      </div>

      {error && (
        <div className="error">
          {error}
        </div>
      )}

      <textarea
        value={input}
        onChange={(e) => setInput(e.target.value)}
        placeholder="Type or speak your message..."
      />

      <button
        onClick={toggleListening}
        disabled={isProcessing}
        className={isListening ? 'recording' : ''}
      >
        {isListening ? '⏹️ Stop' : '🎤 Record'}
      </button>
    </div>
  );
}
```

### With State Indicators

```tsx
import { useVoiceInput } from '@/app/hooks/use-voice-input';
import { useState } from 'react';

function VoiceInputWithStates() {
  const [messages, setMessages] = useState<string[]>([]);

  const {
    isListening,
    isProcessing,
    transcript,
    isSupported,
    mode,
    toggleListening,
    resetTranscript,
    error
  } = useVoiceInput({
    onTranscript: (text) => {
      setMessages(prev => [...prev, text]);
      resetTranscript();
    },
    language: 'en-US'
  });

  if (!isSupported) {
    return (
      <div className="error">
        Voice input is not supported in your browser.
      </div>
    );
  }

  return (
    <div>
      <div className="mode-indicator">
        Mode: {mode === 'gemini' ? '🌟 Gemini (High Accuracy)' : '🌐 Browser (Local)'}
      </div>

      {error && <div className="error">{error}</div>}

      <div className="interim-transcript">
        {isListening && transcript && (
          <p>Interim: {transcript}</p>
        )}
      </div>

      <button
        onClick={toggleListening}
        disabled={isProcessing}
        className={`
          ${isListening ? 'listening' : ''}
          ${isProcessing ? 'processing' : ''}
        `}
      >
        {isListening && '🔴 Recording...'}
        {isProcessing && '⚙️ Processing...'}
        {!isListening && !isProcessing && '🎤 Start Recording'}
      </button>

      <div className="messages">
        {messages.map((msg, i) => (
          <div key={i} className="message">
            {msg}
          </div>
        ))}
      </div>
    </div>
  );
}
```

### Multi-language Support

## Browser Compatibility

### Gemini Mode (MediaRecorder)

- ✅ Chrome 49+
- ✅ Firefox 25+
- ✅ Safari 14.1+
- ✅ Edge 79+

### Native Mode (Web Speech API)

- ✅ Chrome 25+
- ✅ Edge 79+
- ⚠️ Safari (limited support)
- ❌ Firefox (no support)

<Note>
The hook automatically detects capabilities and chooses the best available mode for each browser.
</Note>

## Configuration

### Audio Format Detection

The hook automatically detects the best supported audio format:

1. **webm with opus** (preferred) - Best compression and quality
2. **webm with mp3** - Fallback for Safari
3. **audio/webm** - Generic fallback

### Timeout Settings

The hook handles various timeout scenarios:

- Auto-cancel after network timeout
- Graceful degradation on API errors
- Automatic cleanup on component unmount

## Advanced Features

### Voice Command Integration

Combine with voice command parsing:

```tsx
import { useVoiceInput } from '@/app/hooks/use-voice-input';
import { executeVoiceCommand } from '@/app/actions';

function VoiceCommands() {
  const { toggleListening, isListening } = useVoiceInput({
    onTranscript: async (text) => {
      // Parse voice command
      const result = await executeVoiceCommand(text);
      
      if (result.success && result.command) {
        console.log('Command:', result.command);
        // Execute command...
      }
    },
    language: 'es-ES'
  });

  return (
    <button onClick={toggleListening}>
      {isListening ? 'Listening...' : 'Speak Command'}
    </button>
  );
}
```
