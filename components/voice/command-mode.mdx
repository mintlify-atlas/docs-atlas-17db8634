## Overview

The `VoiceCommandMode` component provides a complete end-to-end voice command workflow. It integrates voice recording, command parsing, user confirmation, and command execution into a single, cohesive interface.

## Features

- **Complete Workflow**: Records voice → Parses command → Shows preview → Executes action
- **Smart Parsing**: Uses AI to extract structured data from voice transcriptions
- **User Confirmation**: Shows parsed command details before execution
- **Error Handling**: Displays parsing and execution errors with retry options
- **Status Feedback**: Real-time status indicators throughout the process
- **Flexible Integration**: Customizable callbacks for different use cases

## Workflow Steps

<Steps>
  <Step title="Voice Recording">
    User clicks VoiceButton to start recording. Component displays "Escuchando..." status.
  </Step>
  
  <Step title="Audio Processing">
    Audio is transcribed using Gemini AI or native browser API. Shows "Procesando comando..." status.
  </Step>
  
  <Step title="Command Preview">
    Parsed command is displayed with extracted fields (equipment, location, priority) for user confirmation.
  </Step>
  
  <Step title="Execution">
    User confirms and command is executed. Status indicator shows progress and result.
  </Step>
</Steps>

## Usage Examples

### Basic Implementation

```tsx
import { VoiceCommandMode } from '@/app/components/features/voice/voice-command-mode';

function WorkOrderPage() {
  const handleCommandExecuted = (result) => {
    console.log('Command executed:', result.message);
    if (result.resourceId) {
      // Navigate to created resource
      router.push(`/work-orders/${result.resourceId}`);
    }
  };

  const handleError = (error) => {
    toast.error(error);
  };

  return (
    <VoiceCommandMode
      onCommandExecuted={handleCommandExecuted}
      onError={handleError}
    />
  );
}
```

## Command Types

The component supports multiple command types:

### Work Order Commands

Creates maintenance work orders with extracted fields:

```typescript
type WorkOrderCommand = {
  type: 'work_order';
  equipment?: string;      // Equipment name or ID
  location?: string;       // Physical location
  priority?: 'urgent' | 'normal' | 'low';
  description?: string;    // Issue description
  assignee?: string;       // Person to assign to
  confidence: number;      // Parse confidence (0-1)
}
```

**Example voice input**: "Crear orden de trabajo urgente para bomba B-102 en planta norte"

### Navigation Commands

Navigates to different screens:

```typescript
type NavigationCommand = {
  type: 'navigation';
  screen?: string;         // Screen name
  path?: string;           // Route path
  confidence: number;
}
```

**Example voice input**: "Ir a historial de órdenes"

### System Commands

Performs system actions:

```typescript
type SystemCommand = {
  type: 'system';
  action: 'theme_mode' | 'logout';
  confidence: number;
}
```

**Example voice input**: "Cambiar tema a oscuro"

## Integration Example

```tsx
import { VoiceCommandMode } from '@/app/components/features/voice/voice-command-mode';
import { useRouter } from 'next/navigation';
import { toast } from 'sonner';

function MaintenancePage() {
  const router = useRouter();

  return (
    <div className="flex flex-col gap-4">
      <h1>Maintenance Dashboard</h1>
      
      <VoiceCommandMode
        minConfidence={0.75}
        context="Maintenance dashboard - Work orders page"
        onCommandExecuted={(result) => {
          toast.success(result.message);
          if (result.resourceId) {
            router.push(`/work-orders/${result.resourceId}`);
          }
        }}
        onError={(error) => {
          toast.error(`Voice command failed: ${error}`);
        }}
        className="fixed bottom-4 right-4 z-50"
      />
    </div>
  );
}
```
