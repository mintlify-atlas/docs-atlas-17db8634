## Environment Variables

GIMA uses environment variables for configuration. All variables are validated at startup using Zod schemas.

### Required Variables

These API keys are **required** for the chatbot to function:

<ParamField path="GROQ_API_KEY" type="string" required>
  GROQ API key for LLaMA 3.3 language model (main chat functionality)
  
  - **Format**: Must start with `gsk_`
  - **Get it**: [console.groq.com/keys](https://console.groq.com/keys)
  - **Free tier**: Yes, generous limits
  
  ```env
  GROQ_API_KEY=gsk_your_api_key_here
  ```
</ParamField>

<ParamField path="GOOGLE_GENERATIVE_AI_API_KEY" type="string" required>
  Google Gemini API key for voice transcription and image analysis
  
  - **Format**: Must start with `AIza`
  - **Get it**: [makersuite.google.com/app/apikey](https://makersuite.google.com/app/apikey)
  - **Free tier**: Yes, 1500 requests/day
  
  ```env
  GOOGLE_GENERATIVE_AI_API_KEY=AIzaYourAPIKeyHere
  ```
</ParamField>

### Optional Variables

<ParamField path="NODE_ENV" type="string" default="development">
  Runtime environment
  
  - **Values**: `development`, `production`, `test`
  - **Effect**: Changes logging, error handling, and security headers
  
  ```env
  NODE_ENV=development
  ```
</ParamField>

<ParamField path="NEXT_PUBLIC_APP_URL" type="string" default="http://localhost:3000">
  Base URL of the application (for production deployments)
  
  ```env
  NEXT_PUBLIC_APP_URL=https://your-domain.com
  ```
</ParamField>

### Feature Flags

Control optional features using these environment variables:

<ParamField path="NEXT_PUBLIC_ENABLE_CHAT_PERSISTENCE" type="boolean" default="false">
  Enable localStorage persistence for chat history
  
  ```env
  NEXT_PUBLIC_ENABLE_CHAT_PERSISTENCE=true
  ```
  
  **When enabled:**
  - Messages persist across page refreshes
  - Stored in browser localStorage with key `gima_chat_history`
  - Users can clear history manually
  
  **When disabled:**
  - Chat history only exists in memory
  - Cleared on page refresh
</ParamField>

<ParamField path="NEXT_PUBLIC_FEATURE_VOICE_COMMANDS" type="boolean" default="false">
  Enable voice-activated work order commands
  
  ```env
  NEXT_PUBLIC_FEATURE_VOICE_COMMANDS=true
  ```
</ParamField>

<ParamField path="NEXT_PUBLIC_FEATURE_PDF_READER" type="boolean" default="false">
  Enable PDF upload and analysis
  
  ```env
  NEXT_PUBLIC_FEATURE_PDF_READER=true
  ```
</ParamField>

### Complete .env.local Example

```env .env.local
# ========================================
# Required API Keys
# ========================================
GROQ_API_KEY=gsk_your_groq_api_key_here
GOOGLE_GENERATIVE_AI_API_KEY=AIzaYourGoogleAPIKeyHere

# ========================================
# Environment Configuration
# ========================================
NODE_ENV=development
NEXT_PUBLIC_APP_URL=http://localhost:3000

# ========================================
# Feature Flags
# ========================================
NEXT_PUBLIC_ENABLE_CHAT_PERSISTENCE=true
NEXT_PUBLIC_FEATURE_VOICE_COMMANDS=true
NEXT_PUBLIC_FEATURE_PDF_READER=true
```

<Warning>
  **Security**: Never commit `.env.local` to version control. It's already in `.gitignore`.
</Warning>

## Troubleshooting

Common issues and solutions:

<AccordionGroup>
  <Accordion title="Installation fails with module not found errors">
    **Symptoms:**
    ```
    Error: Cannot find module '@ai-sdk/react'
    ```

    **Solution:**
    1. Delete `node_modules` and lock file:
       ```bash
       rm -rf node_modules package-lock.json
       npm install
       ```
    2. Ensure you're using Node.js 20+:
       ```bash
       node --version
       ```
  </Accordion>

  <Accordion title="ZodError: Expected string, received undefined">
    **Symptoms:**
    Server crashes on startup with Zod validation errors for environment variables.

    **Cause:**
    Missing or incorrectly formatted API keys in `.env.local`.

    **Solution:**
    1. Verify `.env.local` exists and has correct keys
    2. Check key formats:
       - GROQ: starts with `gsk_`
       - Google: starts with `AIza`
    3. On Windows, ensure UTF-8 encoding (not UTF-16):
       ```powershell
       $content = Get-Content .env.local -Raw -Encoding Unicode
       [System.IO.File]::WriteAllText((Resolve-Path .env.local), $content, [System.Text.UTF8Encoding]::new($false))
       ```
  </Accordion>

  <Accordion title="Chat not responding / Streaming fails">
    **Symptoms:**
    Messages don't appear, or chat shows error state.

    **Solution:**
    1. Check browser console (F12) for errors
    2. Verify API keys are valid:
       - Test GROQ key at [console.groq.com](https://console.groq.com)
       - Test Google key at [AI Studio](https://makersuite.google.com)
    3. Check rate limits (GROQ: 30 req/min)
    4. Restart dev server: `npm run dev`
  </Accordion>

  <Accordion title="Voice input not working">
    **Symptoms:**
    Microphone button does nothing or shows error.

    **Solution:**
    1. Grant browser microphone permission
    2. Check if HTTPS or localhost (WebRTC requires secure context)
    3. Verify `GOOGLE_GENERATIVE_AI_API_KEY` is set
    4. Try fallback: Voice should work with Web Speech API if Gemini fails
  </Accordion>

  <Accordion title="Image analysis fails">
    **Symptoms:**
    Uploading images shows error or no response.

    **Solution:**
    1. Check image size (max 10MB by default)
    2. Verify `GOOGLE_GENERATIVE_AI_API_KEY` is set
    3. Check supported formats: JPEG, PNG, WebP
    4. Check Gemini quota at [AI Studio](https://makersuite.google.com)
  </Accordion>

  <Accordion title="Port 3000 already in use">
    **Symptoms:**
    ```
    Error: Port 3000 is already in use
    ```

    **Solution:**
    Use a different port:
    ```bash
    PORT=3001 npm run dev
    ```

    Or kill the process using port 3000:
    ```bash
    # macOS/Linux
    lsof -ti:3000 | xargs kill

    # Windows
    netstat -ano | findstr :3000
    taskkill /PID <PID> /F
    ```
  </Accordion>

  <Accordion title="TypeScript errors in IDE">
    **Symptoms:**
    Red squiggly lines in VS Code but code runs fine.

    **Solution:**
    1. Restart TypeScript server in VS Code:
       - Cmd/Ctrl + Shift + P
       - "TypeScript: Restart TS Server"
    2. Ensure you opened the project root (not a parent folder)
    3. Check `.vscode/settings.json` for TypeScript config
  </Accordion>

  <Accordion title="Build fails but dev works">
    **Symptoms:**
    `npm run dev` works but `npm run build` fails.

    **Solution:**
    1. Run type-check to find issues:
       ```bash
       npm run type-check
       ```
    2. Fix any TypeScript errors
    3. Check for environment variables used in build (must be set)
    4. Clear Next.js cache:
       ```bash
       rm -rf .next
       npm run build
       ```
  </Accordion>
</AccordionGroup>

## Development Workflow

Recommended workflow for contributing to GIMA:

<Steps>
  <Step title="Setup Git Hooks">
    Husky automatically installs Git hooks for:
    - **Pre-commit**: Lints and formats staged files
    - **Commit-msg**: Validates commit message format
    
    ```bash
    npm run prepare
    ```
  </Step>

  <Step title="Create Feature Branch">
    ```bash
    git checkout -b feature/your-feature-name
    ```
  </Step>

  <Step title="Make Changes">
    - Write code
    - Add tests in `tests/` directory
    - Update documentation if needed
  </Step>

  <Step title="Run Quality Checks">
    ```bash
    # Lint and format
    npm run lint:fix
    npm run format

    # Type check
    npm run type-check

    # Run tests
    npm test
    ```
  </Step>

  <Step title="Commit with Conventional Commits">
    ```bash
    git add .
    git commit -m "feat: add new feature"
    ```
    
    Allowed types: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`
  </Step>

  <Step title="Push and Create PR">
    ```bash
    git push origin feature/your-feature-name
    ```
    
    Then create a Pull Request on GitHub.
  </Step>
</Steps>

<Tip>
  **Commit Message Format**: `type(scope): description`
  
  Examples:
  - `feat(chat): add message reactions`
  - `fix(voice): resolve transcription timeout`
  - `docs(readme): update installation steps`
</Tip>
